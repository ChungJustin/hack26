# Comprehensive Project Plan: Sikgu (식구) - "저희는 식구를 만들어 줍니다"

## 1. Concept (기획 배경)
- **이름:** **Sikgu (식구)** - "함께 밥을 먹는 입"이라는 뜻의 한국어
- **가안:** 1인 가구에서 문제 식생활을 연결로 해결하는 플랫폼
- **문제점:** 다양하지 않은 음식, 높은 배달비, 대용량 식재료 처리의 어려움
- **슬로건:** "저희는 식구를 만들어 줍니다"
- **최종 목표:** 밥친구 3명 만들어주기 (본인 포함 총 4명의 강한 본딩 형성)

## 2. User Flow & Onboarding (앱 시작하기 및 설문)

### Step 1: 니즈 파악 (체크리스트)
- [ ] 시간될 때 같이 밥 먹을 친구가 있으면 좋을 것 같아요 (동성/이성 선택)
- [ ] 배달비를 아끼고 싶어요 (예: 피자 사서 나눠먹기)
- [ ] 음식(반찬)을 한번에 만들어서 나눠먹고 싶어요
- [ ] 식재료를 나누고 싶어요
- [ ] 다양한 음식을 나누고 싶어요

### Step 2: 나의 기여도(Talent) 파악
- [ ] 4인분의 파스타를 만들 수 있어요
- [ ] 한식 반찬을 잘 만들어요
- [ ] 새로운 음식을 만드는데 도전하는걸 좋아해요
- [ ] 집에 오븐이 있어서, 빵이나 쿠키를 자주 만들어 먹어요
- [ ] 식재료를 싸게 잘 사요 (구독 여부: 컬리멤버스/쿠팡프레쉬/이마트/코스트코)
- *기타: 식구를 만들어 같이 해결하고 싶은 문제를 직접 작성*

### Step 3: 위치 인증
- 사는 곳 입력: 단지명 / 오피스텔명 / 건물명 등

## 3. Core Features (실제 UI 구성)

### A. 목적 그룹 리스트 (Main Feed)
- **반찬 나눔:** "반찬 나눠요!"
- **요리/포트럭:** "식재료 나누고 가끔 같이 만들어 먹어요! (이탈리안 위주)"
- **공동 구매:** "같이 코스트코에 장보러 가서 재료 나눠요! (돼지고기 소분 필요)"
- **배달 쉐어:** "족발 / 피자 같이 주문해서 나누기"

### B. 프리셋 기능 (Quick Create)
- 족발/피자 배달 쉐어 프리셋
- 코스트코/대형마트 장보기 쉐어 프리셋
- 반찬 교환 프리셋

### C. 이전 식구 (Bonding History) - "Stretch Goal"
- 같이 식생활을 해결한 사람들 목록 노출
- 예시 데이터: 
  - **코딩의 노예** (이전에 같이 반찬을 나눴어요!)
  - **코딩코딩** (코스트코에서 물건을 같이 샀어요!)
  - **코딩조아**
  - **리눅시아스**

## 4. Implementation Strategy (구현 계획)
- **구현 방식:** 1인 가구의 '연결'을 통한 식생활 해결 (배달비 나누기, 포트럭, 반찬 나눔)
- **UI Focus:** 직관적인 체크리스트 기반 온보딩 + 목적별 그룹핑 리스트
- **Targeting:** 단순 만남을 넘어선 "식구" 관계 형성 (강한 본딩)

## 5. Timeline (5 Hours)
- **H1:** Sikgu 초기 셋업 및 온보딩 설문 시스템 구현 (Step 1~3)
- **H2:** 메인 피드 및 목적 기반 그룹 UI (목적 리스트 3개+)
- **H3:** 프리셋 기반 그룹 생성 로직 구현
- **H4:** '이전 식구' 히스토리 및 본딩 대시보드
- **H5:** 전체 UI 폴리싱 및 발표 준비 (문제정의/사업성 강조)
